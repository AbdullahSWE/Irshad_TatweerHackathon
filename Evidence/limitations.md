# Limitations

This file exists to keep the submission honest.

| Limitation | Current state | Mitigation / next step |
|---|---|---|
| No government submission | Irshad does not submit forms or applications. | Integrate only through official channels if an authority partner is available. |
| Fees can change | Cost output should be treated as estimated unless officially confirmed. | Store source URL, `last_verified`, and confidence label for every fee. |
| KB maintenance required | Regulations and contact details can become stale. | Assign a KB owner and review schedule. |
| Community testing still limited | Hackathon time may limit access to target users. | Run 5–10 tests and fill `evidence/test-runs.md`. |
| Internet required | Prototype uses a backend and LLM call. | Add cached KB and offline draft mode later. |
| Arabic quality depends on voice model | Apple voices are acceptable but not as natural as dedicated TTS. | Swap TTS later if budget and latency allow. |
| Does not guarantee bank approval | Bank requirements vary and change. | Present banks as candidates and list documents / questions to confirm. |
