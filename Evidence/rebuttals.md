# Rebuttals: hard judge questions

| Judge pushback | Answer |
|---|---|
| Why not just use TAMM or ADDED? | Official portals are where the process happens. Irshad helps a first-time founder know what to ask, which licence path might apply, and what documents to prepare before going there. |
| Are you replacing official authorities? | No. Irshad does not issue licences, submit applications, guarantee approval, or replace ADRA / ADDED / ADAFSA / banks. |
| What if the AI is wrong? | Important outputs are labelled verified, estimated, unverified, or missing. Unsupported claims should be shown as unverified with the exact authority question to ask. |
| Is this just a chatbot? | No. The app follows a fixed journey backbone with stage-specific adaptive cards, a completeness gate, structured output, and trust labels. |
| Who maintains it? | A local operator, incubator partner, or authority partner can update the JSON knowledge base and source matrix. The iOS app does not need resubmission for every guidance update. |
| Can it scale? | The reusable engine stays the same; the local knowledge base and archetypes change per community / emirate. |
| What is the biggest risk? | Stale or incomplete official data. We address this by showing source dates, maintaining a source matrix, and clearly marking unverified items. |
| What would make it production-ready? | Official data/API partnerships, stronger user testing, review workflow for KB updates, privacy review, and offline/cached mode. |
