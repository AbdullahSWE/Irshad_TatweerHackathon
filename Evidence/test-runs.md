## What to capture

- Start screen or spoken input.
- One adaptive question card.
- Final plan or unverified state.
- Number of questions.
- Whether the app clearly labelled uncertainty.

## Pass criteria

| Criterion | Pass condition |
|---|---|
| Adaptive questions | Different archetypes produce visibly different question content. |
| Completion | A supported persona reaches a final plan. |
| Grounding | Unsupported facts are labelled unverified or missing. |
| Usability | User can understand the next action without reading code. |
